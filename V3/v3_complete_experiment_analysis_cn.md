# V3 完整实验分析

## 1. 总体结论

V3 实验已经完成从上游联邦训练、下游 RAG 评测到全局归档报告的完整闭环。总体结果表明，V3 相比 V2 的主要改进方向是有效的：在主结果、预算实验和异构性实验中，`hypernet_v3` 都能够在相同或更低预算下实现更低的平均上传比例，并且在当前评测设置下没有观察到明显的下游性能崩溃。

换言之，V3 已经不再只是证明“超网络选择性上传可以工作”，而是进一步证明了“引入客户端条件特征、历史记忆特征和 value-aware 自适应预算后，通信压缩效果还能继续提高”。

## 2. 主结果分析：V3 相比 V2 的直接收益

在 `v3_main` 中：

- `full` 的平均上传比例为 `1.0000`
- `random(k=3)` 为 `0.2575`
- `delta_norm(k=3)` 为 `0.2377`
- `hypernet_v2(k=3)` 为 `0.2316`
- `hypernet_v3(k=3)` 为 `0.2259`

这说明：

1. `hypernet_v3` 在主结果实验中取得了最优平均上传比例。
2. V3 相比 V2 的提升幅度不算极大，但在多 seed 下稳定存在，说明改进并非偶然。
3. 相比 `random` 和 `delta_norm`，V3 至少已经清楚地证明自己不是“更复杂但没有收益”，而是在通信压缩层面提供了真实增益。

因此，V3 的第一层结论可以概括为：**V3 已经把 V2 的上游通信效率进一步向前推进了一步。**

## 3. 预算实验分析：V3 的价值在低预算区间更明显

`v3_budget` 是本轮最关键的一组实验，因为它直接回答“在不同通信预算下谁更值”。

### 3.1 低预算区间

在 `k=1` 时：

- `random`：`0.1095`
- `delta_norm`：`0.1073`
- `hypernet_v2`：`0.1073`
- `hypernet_v3`：`0.1020`

这说明在极低预算区间，`hypernet_v3` 已经实现了最强压缩，平均压缩率接近 `89.8%`。  
与 `random` 相比，V3 进一步减少了通信；与 `delta_norm` 和 `hypernet_v2` 相比，V3 也略优。

### 3.2 中预算区间

在 `k=3` 时：

- `random`：`0.2575`
- `delta_norm`：`0.2431`
- `hypernet_v2`：`0.2316`
- `hypernet_v3`：`0.2263`

这说明在主工作区间里，V3 的优势依然稳定存在。  
从论文角度看，这一组是最适合作为“默认推荐配置”的，因为它既不像 `k=1` 那样极端，也已经明显体现了优于基线的通信表现。

### 3.3 高预算区间

在 `hypernet_v3, k=5` 时，平均上传比例为 `0.3489`。  
这进一步说明 V3 已经形成了清晰的预算-压缩曲线：`k` 越大，通信开销越高，但依然保持比全量上传显著更低的比例。

综上，`v3_budget` 的意义在于：**V3 不只是“能压缩”，而是已经能提供多预算点可比较的结果曲线。**

## 4. 异构性实验分析：V3 在更强 non-IID 条件下保持稳定

在 `v3_heterogeneity` 中，分别考察了 `alpha=0.5 / 0.3 / 0.1` 三档异构程度。

### 4.1 `random` 的表现

`random` 在三个场景中都维持在 `0.2575` 左右，说明它本身不显式感知异构性。

### 4.2 `delta_norm` 的表现

`delta_norm` 在 `alpha=0.1` 时上升到 `0.3042`，相比 `alpha=0.5 / 0.3` 的 `0.24` 左右明显变高，说明在更强异构场景下，基于局部更新强度的启发式选择会出现额外通信波动。

### 4.3 `hypernet_v2` 与 `hypernet_v3`

- `hypernet_v2` 在三档异构程度下基本稳定在 `0.2316`
- `hypernet_v3` 分别为：
  - `alpha=0.5`: `0.2244`
  - `alpha=0.3`: `0.2263`
  - `alpha=0.1`: `0.2256`

这说明 V3 至少体现出两点：

1. 在更强 non-IID 条件下，它仍然稳定。
2. 它始终保持低于 `hypernet_v2` 的上传比例。

因此，本轮异构性实验支持这样一个结论：**V3 在更异构的联邦场景下具备稳定性，并保持了相对于 V2 的通信优势。**

## 5. 特征消融分析：value-aware 和客户端信息是有效的

`v3_ablation_feature` 的结果很有启发性。

### 5.1 score_mode 的影响

- `importance` 打分：`0.2345`
- `value` 打分：`0.2263`

这表明把目标从单纯 importance 升级为 value-aware 确实带来了通信收益。  
也就是说，V3 的收益不是仅靠“超网络更复杂”得来的，而是因为它更接近“收益/成本”视角。

### 5.2 client embedding 的影响

- 保留 client embedding：`0.2263`
- 去掉 client embedding：`0.2299`

差异不算非常大，但方向明确，说明客户端条件特征是有帮助的。

### 5.3 history features 的影响

- 保留 history：`0.2263`
- 去掉 history：`0.2233`

这个结果比较有意思：在当前设置下，去掉 history 反而略低。  
这说明当前历史特征还没有完全转化成显著收益，或者在现有任务难度下，历史记忆的价值还没有被充分激发。

因此，特征消融可以形成较为稳妥的分析结论：

- `value-aware` 是当前 V3 提升最明确的来源。
- `client embedding` 提供了小幅正向帮助。
- `history features` 的收益还不够稳定，是后续 V4 或方法增强的重点。

## 6. 预算机制消融分析：自适应预算的潜力还未完全释放

`v3_ablation_budget` 中：

- `adaptive`：`0.2263`
- `fixed`：`0.2263`

两者当前结果几乎相同。  
这说明：

1. 当前自适应预算机制至少没有把结果做坏。
2. 但它的额外收益还没有显著体现出来。

从 `avg_predicted_budget_ratio_mean = 0.4838` 可以看出，自适应预算头在学习预算信号；但在当前实现和任务难度下，这一学习还没有转化成比固定预算更好的最终上传结果。

因此，这部分更适合解释为：**自适应预算是一个合理但尚未完全释放收益的设计。**

## 7. 下游评测分析：整体稳定，但区分度有限

从 `full_pipeline_all_v3` 看，已完成下游实验数与上游实验数一致，说明全链路已经补齐。  
但是当前下游结果呈现出一个明显现象：大量配置的 `cos_3 / recall_3 / mrr / NDCG` 非常接近，主要集中在：

- `cos_3 ≈ 0.94`
- `recall_3 ≈ 0.9133`
- `mrr ≈ 0.9000`
- `NDCG ≈ 0.8855`

只有少数运行略高，比如：

- `full_k0_w1_s0`：`NDCG ≈ 0.9132`
- `random_k3_w1_s0`：`NDCG ≈ 0.9058`
- `hypernet_v3_k3_w1_s0`：`NDCG ≈ 0.8958`

这说明当前下游评测对不同上传策略的区分度仍然偏弱。  
因此，当前阶段更适合得出的结论是：

- V3 在下游层面没有出现明显崩溃。
- 但下游指标还不足以像上游通信结果那样清楚地区分方法优劣。

也就是说，本轮 V3 更强的证据仍然主要来自上游通信压缩，而不是下游显著提升。

## 8. 结果中的重要限制

本轮结果有两个必须明确说明的限制。

### 8.1 `v3_ablation_budget` 有效输出数少于配置数

suite 报告里是 9 个配置，但最终有效上游 run 为 6。  
这意味着预算机制消融的结论应解释为“趋势性结论”，而不是非常严格的统计结论。

### 8.2 下游结果区分度不足

虽然下游评测已经补齐，但很多配置的指标非常接近，这会削弱“V3 在 end-to-end 指标上显著优于基线”的论证力度。

## 9. 最终总结

综合全部 V3 结果，可以形成如下结论：

1. **V3 在上游通信压缩上已经稳定优于 V2。**
2. **`v3_budget` 表明 V3 尤其在低预算和中预算区间更具优势。**
3. **`v3_heterogeneity` 表明 V3 在更强 non-IID 场景下依然稳定，并保持通信优势。**
4. **`value-aware` 是目前最明确有效的改进点。**
5. **自适应预算和历史特征机制已经合理接入，但额外收益还未完全显现。**
6. **下游结果整体稳定，但当前评测设置对方法差异的区分度仍有限。**

因此，本轮 V3 的最稳妥结论是：

**V3 已经成功把研究重点从“选择性上传是否可行”推进到了“如何在相同预算下更高效地使用通信资源”；其最大亮点是稳定改善了上游通信效率，而下一步最值得加强的是下游区分度和预算机制收益的进一步放大。**
